Boston Airbnb Fair Pricing Tool and Recommender


Developed a pricing regression model to estimate Boston Airbnb listing prices. Used web scraping, APIs, and natural language processing (NLP) techniques to engineer features. Implemented a recommender system using cosine similarity to suggest listings based on user inputs. The pricing model outperformed the null model by reducing the root mean squared error from $86 to $49 and achieved an R-Squared value of 0.68 on validation data with outliers removed.

Key Accomplishments
  • Data were collected from Inside Airbnb for over 3800 Airbnb listings in the Boston Area in 2023 with 75 features provided for each listing.
  • A feature to simulate the distance of each listing to Boston subway stations was created by web scraping station addresses from the Massachusetts Bay Transit Authority (MBTA) website before using the Google Geocoding API to return latitude and longitude coordinates for each listing.
  • Additional features were engineered which used Natural Language Processing (NLP) Techniques to examine descriptive text columns.
  • The correlation coefficients between all features and listing price were used to select the best features on which to train the model.
  • Seven different regression models were run, and the best models were optimized using grid searching. Principal Component Analysis (PCA) was also attempted to reduce dimensionality and improve price predictions.
  • A simple recommender employing cosine similarity was created to find listings most similar to simulated user-input listing criteria.
Key Outcomes
  • Outliers had a large impact on the model performance with the best models initially achieving a root mean squared error (RMSE) of $389 and an R-Squared value of 0.09 on validation data.
  • Performance improved drastically by removing listings with prices below the 5th percentile and above the 95th percentile to an RMSE of $49 and an R-Squared value of 0.68. This is an improvement over the null modelâs RMSE of $86 with outliers removed.
  • The best-performing model was a random forest regressor.
  • A recommender was capable of correctly recommending listings that met the simulated user-input listing criteria.

Identifying Humor with Language Models


Created classification models employing natural language processing to categorize news article titles collected from the r/worldnews and r/TheOnion subreddits using the Pushshift API. Regular expressions were used to clean the data, removing errors and subreddit identifiers in the titles. A variety of classification models were attempted and evaluated. The best-performing model was a stacking classifier with an accuracy of 86.0% on unseen data.

...Or click here to expand a summary!
Key Accomplishments
  • The Pushshift API was used to collect approximately 6000 news article titles from both the r/worldnews and r/TheOnion subreddits.
  • Natural language processing (NLP) techniques such as tokenization, stop word removal, and sentiment analysis were employed as pre-modeling and exploratory data analysis (EDA) steps.
  • Of seven different classification models initially attempted, the three best models were selected for hyperparameter tuning (optimization) using grid searching.
  • The three best-optimized models were then incorporated into a stacking classifier ensemble model (using a logistic regression model as the final estimator) to produce the best results.
Key Outcomes
  • The final model achieved an accuracy of 86% on unseen data, 89% on testing data, and 99.99% on training data. This is an improvement over the null model accuracy of approximately 51%.
  • It was speculated that the different subject matter of the two subreddits was likely the reason for such a high accuracy and not due to the modelâs ability to correctly identify humor.


Home Price Estimator for Ames, Iowa


Built a linear regression model to predict the price of homes in Ames, Iowa based on home and property characteristics. Engineered interaction and polynomial features to increase model accuracy. The best model achieved an R-Squared value of 0.89 and was able to predict home prices within a mean absolute error of $18,643 and a root mean squared error of $25,887 on testing data.

Key Accomplishments
  • Features were selected based both on their correlations to the home price and common knowledge of important home characteristics that would influence the sale price or value.
  • Different sets of features were run through a linear regression model to find features that would produce the best model performance.
  • Interaction terms and polynomial feature transforms were created and included in the prediction model based on their correlations to price.
  • Regularization techniques were attempted to improve model performance.
Key Outcomes
  • The final model that was selected was trained on a dataset that included polynomial features with a mean absolute error of $18,643 - well within the $30,000 goal - and an improvement on the null model RMSE of $78,827.
  • Regularization methods produced a slight reduction in model performance with increases in the RMSE below or around 1% of the RMSE from the non-regularized model.


Carbon Emissions Forecasting


Forecasted carbon emissions levels from natural gas electric energy production one year beyond the provided historical time series data. Optimized a variety of autoregression analysis models with hyperparameter tuning, selected the type of transformation for the training data, and chose the optimal time span of historical data to be used to train the model. The best model achieved an estimated mean absolute percent error of 3%.

  • Different data transformation techniques were evaluated to optimize model performance and various training data time spans were examined to reduce prediction error and determine how much of the provided training data should be used to train the model.
  • AR, MA, ARMA, ARIMA, and SARIMA models were optimized and evaluated.
